Process Scheduling Simulator

The Process Scheduling Simulator is an application designed to visualize and simulate various process scheduling algorithms. The simulator provides a dynamic way to understand how different scheduling methods manage processes in a system. The main features of this simulator include:

  • Real-Time and Instant Simulations: The simulator supports both real-time and instant simulations, allowing users to observe the process scheduling over time or view the final state immediately.
  • Algorithms Supported: The simulator includes multiple scheduling algorithms:
    • First-Come, First-Served (FCFS): Processes are scheduled in the order they arrive.
    • Priority Scheduling (Non-Preemptive): Processes are scheduled based on priority without interruption.
    • Priority Scheduling (Preemptive): Processes with higher priority can interrupt lower priority processes.
    • Round-Robin (RR): Processes are assigned time slices in a cyclic order.
    • Shortest Job First (SJF, Non-Preemptive): Processes with the shortest execution time are scheduled first.
    • Shortest Job First (SJF, Preemptive): Shorter jobs can preempt longer ones.
    • Feedback Scheduling (Feedback): Processes move between queues of different priority levels based on their execution.
  • Operation Management: Each operation within the simulator has designated properties, including priority, time left for execution, ID, arrival time, response time, and execution time.
  • Queue Management: The simulator uses a queue interface to manage and operate on the operations in unit time orientation. Key functions include enqueuing operations, consuming time units, iterating existing operations, and cloning queues.
  • User-Friendly Features:
    • Error Handling: The simulator handles errors, such as invalid input for arrival, burst, and priority times, by alerting the user.
    • Dynamic Process Management: Users can pause, add, remove, and restart processes dynamically during the simulation.
    • Resizable Window: The window can be resized without losing the simulation state, providing flexibility and better visibility.
    • Advanced Control Options: The simulator allows users to remove multiple operations at once and provides warnings for invalid actions (e.g., attempting to remove a row without selection).
